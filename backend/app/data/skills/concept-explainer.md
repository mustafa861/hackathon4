# Skill: Concept Explainer

## Metadata
- **Name:** concept-explainer
- **Description:** Explain concepts at various complexity levels
- **Trigger Keywords:** "explain", "what is", "how does", "tell me about", "describe"

## Purpose
Break down complex concepts into understandable explanations tailored to the learner's level. Provide clear, accurate explanations with relevant examples and analogies.

## Workflow
1. **Identify the concept** the student is asking about
2. **Assess the student's level** based on their question and progress
3. **Choose appropriate complexity:**
   - Beginner: Simple language, everyday analogies, no jargon
   - Intermediate: Technical terms with definitions, practical examples
   - Advanced: Deep technical detail, edge cases, best practices
4. **Structure the explanation:**
   - Start with a simple definition
   - Provide an analogy or real-world example
   - Explain how it works
   - Show why it matters
   - Connect to related concepts
5. **Check understanding** by asking if they'd like more detail or a different explanation

## Response Templates

### Beginner Level
```
Great question! Let me explain [concept] simply:

[Simple definition in everyday language]

Think of it like [everyday analogy]...

Here's why this matters: [practical importance]

Would you like me to go deeper, or shall we move to an example?
```

### Intermediate Level
```
[Concept] is [technical definition].

Here's how it works:
1. [Step 1]
2. [Step 2]
3. [Step 3]

Example:
[code or practical example]

Key things to remember:
- [Key point 1]
- [Key point 2]

Want to dive into any specific part?
```

### Advanced Level
```
[Concept] at a deep level involves [technical explanation].

Implementation details:
[Detailed technical breakdown]

Edge cases and considerations:
- [Edge case 1]
- [Edge case 2]

Best practices:
- [Best practice 1]
- [Best practice 2]

Related concepts: [connections to other topics]
```

## Key Principles
- Always ground explanations in the course content
- Use analogies that relate to things students already know
- Never assume knowledge not covered in previous chapters
- Encourage questions and offer alternative explanations
- Keep explanations concise - offer to go deeper if needed
- Use the student's progress to gauge appropriate complexity
